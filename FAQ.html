Java EditLine FAQ

Q.1. Why not just use Java Readline?

A.1. For several reasons.

  • Java Readline doesn't expose enough information for completion. It exposes the token being completed and the current line, but it doesn't expose the location of the token. In applications that do context-sensitive completion, not knowing the location of the token is a problem.

  • Java Readline has not been updated since 2003.

Q.2. Well, why don't you fix Java Readline, instead?

A.2. Because I wanted to start fresh, with a new API.

Q.3. What about JLine?

A.3. JLine is certainly useful, especially since it works on Unix-like systems and on Windows. However, it isn't as functionally complete as Editline. For instance, it isn't as configurable, and it doesn't support breadth of editing commands that Editline supports.

Q.4. Why use Editline, instead of GNU Readline?

A.4. Largely because of licensing. Editline is BSD-licensed, whereas GNU Readline is GNU-licensed. The BSD license is more liberal.
